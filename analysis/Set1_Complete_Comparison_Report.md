## 四、方法对比详情

### 4.1 Direct LLM Baseline

**特点**:
- 直接使用LLM生成报告，无外部知识检索
- 依赖模型的参数知识（pre-trained knowledge）
- 无PubMed/OncoKB/指南引用

**量化表现**:
- CPI: 0.496（最低）
- PTR: 0.050（几乎无引用）
- CER: 0.148（存在Major错误）

**主要问题**:
1. 临床决策缺乏证据支撑
2. 剂量参数多为估计值，无文献支持
3. 无排他性论证（Rejected Alternatives）
4. Module 6适用性判断经常错误

### 4.2 RAG Baseline

**特点**:
- 使用向量检索从指南文档获取相关知识
- 基于检索内容生成报告
- 有本地指南引用但格式不规范

**量化表现**:
- CPI: 0.719
- PTR: 0.524（中等）
- CCR: 3.089（较好）

**主要问题**:
1. 检索结果与问题匹配度不稳定
2. 引用格式不统一（非标准[Local: ...]格式）
3. PubMed/OncoKB引用缺失
4. 报告结构化程度不足

### 4.3 WebSearch LLM Baseline

**特点**:
- 使用网络搜索获取最新信息
- 有网络来源但未格式化为标准引用
- 报告质量较高但可追溯性不足

**量化表现**:
- CPI: 0.718（与RAG相当）
- CCR: 3.333（CCR最高）
- CER: 0.000（无错误）

**主要问题**:
1. 搜索来源未格式化为标准引用标签
2. 网络信息质量参差不齐
3. 无PubMed PMID标注
4. 响应时间较长（112s vs BM Agent ~180s）

### 4.4 BM Agent

**特点**:
- 自主规划+多工具调用（OncoKB、PubMed、指南检索）
- 每个声明都有标准格式引用
- 8模块完整结构，Module 6动态判断

**量化表现**:
- CPI: 0.948（最高）
- PTR: 0.990（接近完美）
- CER: 0.000（无错误）
- MQR: 3.867（优秀）

**核心优势**:
1. **Deep Drill Protocol**: 强制PubMed检索后才可使用免责声明
2. **引用审计**: submit_mdt_report自动验证引用完整性
3. **5个强制检查点**: 确保治疗线、排他性论证、剂量验证等
4. **多源证据整合**: PubMed + OncoKB + 本地指南三重验证

## 七、结论与建议

### 7.1 主要结论

1. **BM Agent在Set-1批次测试中表现卓越**:
   - CPI 0.948，显著优于所有Baseline（0.496-0.719）
   - PTR 0.990，接近完美可追溯性
   - CER 0.000，无临床错误

2. **Baseline方法局限性明显**:
   - Direct LLM: 无证据支撑，PTR≈0
   - RAG: 检索质量不稳定，引用格式不规范
   - WebSearch: 来源可追溯性不足

3. **公平性确认**:
   - 所有方法使用相同Set-1输入文件
   - 无信息泄露问题
   - 对比结果可信

### 7.2 临床适用性评估

| 方法 | 临床可用性 | 原因 |
|------|-----------|------|
| BM Agent | ✅ **推荐使用** | PTR>0.95, CER=0, 所有声明可验证 |
| WebSearch | ⚠️ 可用但需审核 | CER=0但PTR低，需人工验证来源 |
| RAG | ⚠️ 需大量审核 | PTR中等，引用格式不规范 |
| Direct LLM | ❌ **不推荐** | PTR≈0，无证据支撑，存在Major错误 |

### 7.3 未来改进建议

**For Baseline**:
1. 强制要求PubMed引用（如BM Agent的Deep Drill Protocol）
2. 统一引用格式标准
3. 添加Module 6适用性判断逻辑

**For BM Agent**:
1. 继续优化RARE_MUTATION_DEEP_DRILL流程
2. 减少患者640880类首次PubMed调用为0的情况
3. 保持CPI>0.90的高标准

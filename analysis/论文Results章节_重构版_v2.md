# 4.4 实验结果与分析（重构版）

## 4.4 实验结果与分析

为系统性验证基于Agent Harness与认知执行协议的肿瘤脑转移MDT智能体（BM Agent）的有效性，本节在真实世界临床队列上开展了严格的对抗实验。实验从诊断效能与资源消耗的双重视角，对四种技术范式进行了深度剖析。

### 4.4.1 数据集构建与基准方法设置

#### 4.4.1.1 真实世界脑转移多模态数据集

鉴于开源界缺乏针对复杂肿瘤MDT的标准评测集，本研究从真实临床环境中提取并结构化了9例极具挑战性的肿瘤脑转移患者病历（所有患者敏感信息均已经过脱敏处理，患者ID已做哈希处理）。

该数据集具有以下三个显著的复杂性特征：

1. **多模态与时间跨度长**：涵盖了患者的原发灶确诊、多线全身系统性治疗（最多六线治疗）、以及新发脑转移的纵向演进过程。病例包含初治患者（Case 9）至六线治疗后进展患者（Case 8）。

2. **基因组学异质性高**：病例中包含了不同的分子病理分型，包括KRAS G12V（Case 8）、EGFR突变（Case 7）、HER2阳性（Case 10）、MET扩增（Case 9）、SMARCA4突变（Case 3）等，要求系统必须动态检索靶向药物的最新循证等级。

3. **干预手段的冲突与权衡**：所有测试病例均处于治疗的决策节点，需要系统在"立体定向放射外科（SRS）"、"全脑放疗（WBRT）"、"开颅手术切除"与"系统治疗"之间进行复杂的排他性论证。

#### 4.4.1.2 对比基准（Baselines）设置

为全面评估本系统的先进性，实验选取了当前医疗人工智能领域主流的三种技术架构作为基准对比方法。所有方法均在统一的模型底层（Qwen-Plus，Temperature=0.3, Top-P=0.95）上运行。

**Baseline 1：Direct LLM（直接大语言模型零样本生成）**

设置：代表最基础的AI应用范式。仅将患者的病历文本与基础Prompt输入给大模型，不允许其调用任何外部工具或知识库，完全依赖其预训练参数权重生成MDT报告。

**Baseline 2：Direct LLM + Web Search（基于通用网络搜索的LLM）**

设置：在单体大模型的基础上赋予其开放域的网络搜索能力。系统可以通过搜索引擎（DuckDuckGo）获取互联网信息以补充医学知识。搜索策略为自动触发式：当模型判断需要外部信息时，自动发起2-3次搜索查询。此基准旨在验证"无约束的宽泛检索"是否会引入上下文污染与非权威证据。

**Baseline 3：Standard RAG（标准检索增强生成）**

设置：传统的医疗AI范式。系统本地挂载了NCCN/ASCO等权威脑转移指南的向量数据库（使用text-embedding-3-small嵌入）。在生成报告前，系统首先通过计算向量相似度检索出Top-5的指南切片片段（Chunks，每块约500 tokens），并将其作为上下文拼接入提示词中辅助生成。

**Ours: Agent Harness + Deep Drill SKILLs（本研究提出的智能体架构）**

设置：采用第4.3节设计的扁平化单一主控架构，挂载完整的四大核心技能（PDF视觉探针、OncoKB基因库、PubMed动态文献检索、MDT结构化认知协议），并强制开启五大检查点与深钻溯源机制。

### 4.4.2 结构化多维量化评估体系

在医疗AI研究中，脱离临床实践的纯文本相似度指标（如BLEU、ROUGE）无法反映真实临床价值。MDT报告的核心在于其结论的安全性、合规性与可执行性。基线模型往往只输出单薄的段落，而本系统输出8模块结构化报告，这给对齐评估带来了挑战。

为此，本研究设计了"临床专家双盲评审结合LLM辅助提取"的评估流水线。评估由资深肿瘤内科与神经外科医师独立进行，分歧案例经MDT讨论后确定最终评分。评估体系包含以下核心维度：

#### 4.4.2.1 临床决策一致性准确率（Clinical Consistency Rate, CCR）

**临床动机**：MDT的首要目标是给出正确的治疗方向。模型输出的核心干预推荐必须与真实世界中顶级三甲医院专家团队的最终定论（Ground Truth）高度一致。

**评估逻辑**：评审专家独立提取各方法的治疗线判定、首选方案、剂量范围与排他性论证。采用4分制并解构为加权公式：

$$CCR = 0.40 \times S_{line} + 0.35 \times S_{scheme} + 0.15 \times S_{dose} + 0.10 \times S_{reject}$$

其中，$S_{line}$为治疗线判定准确性（0-2分），$S_{scheme}$为方案匹配度（0-2分），$S_{dose}$为剂量参数准确性（0-2分），$S_{reject}$为排他性论证充分性（0-2分）。总分经归一化至0-4分。

#### 4.4.2.2 物理可追溯率（Physical Traceability Rate, PTR）

**临床动机**：在医疗实践中，"证据主权"高于一切。医生无法采信大模型凭借预训练权重"脑补"出的处方剂量。该指标专门用于量化本系统"深钻机制（Deep Drill）"阻断参数幻觉的效果。

**评估逻辑**：评审专家提取报告中所有的医疗参数声明（总数为$N$），并强制启动独立的二次API检索，核查其引用的指南行号或PMID是否真实存在且数据吻合。每条引用的可追溯性评分定义如下：

$$TraceScore(i) = \begin{cases} 2 & \text{完全可追溯（标准格式+物理验证通过）} \\ 1 & \text{部分可追溯（有引用但格式不规范）} \\ 0 & \text{不可追溯（无引用或虚假引用）} \end{cases}$$

$$PTR = \frac{\sum_{i=1}^{N} TraceScore(i)}{2N} \in [0, 1]$$

#### 4.4.2.3 MDT报告规范度（MQR）与临床错误率（CER）

**临床动机与评估逻辑**：

- **规范度（MQR）**：考察8大模块（入院评估、原发灶处理、系统管理、随访方案、排他性论证、围手术期管理、分子病理、执行路径）的完整性。评审引入"动态N/A（不适用）评分机制"：如对无手术指征患者，Module 6正确输出N/A即得满分。评分公式：

$$MQR = 0.40 \times S_{complete} + 0.30 \times S_{applicable} + 0.20 \times S_{structure} + 0.10 \times S_{uncertainty}$$

- **错误率（CER）**：安全底线指标。依据错误致害性分级：严重错误（权重3.0，如推荐禁忌方案）、主要错误（权重2.0，如剂量计算错误>20%）、次要错误（权重1.0，如随访间隔不当）。

$$CER = \frac{\sum_{j} (w_j \times n_j)}{W_{max}} \in [0, 1]$$

**综合性能指数（CPI）**：为实现全局对比，聚合上述指标构建加权综合指数：

$$CPI = 0.35 \times \left(\frac{CCR}{4}\right) + 0.25 \times PTR + 0.25 \times \left(\frac{MQR}{4}\right) + 0.15 \times (1 - CER)$$

权重设定依据临床优先级：CCR（临床一致性）占35%，PTR（证据可靠性）与MQR（报告质量）各占25%，CER（错误惩罚）占15%。

### 4.4.3 综合诊断效能与"零幻觉"溯源分析

在9例高度复杂的真实患者测试中，实验数据充分证实了Agent Harness架构的显著优势（表5）。

**表5 BM Agent与基线方法的量化对比**

| Method | CCR (0-4) | PTR (0-1) | MQR (0-4) | CER (0-1) | CPI (0-1) |
|:-------|:---------:|:---------:|:---------:|:---------:|:---------:|
| Direct LLM | 1.5 ± 0.2 | 0.00 ± 0.00 | 2.0 ± 0.2 | N/A | 0.42 ± 0.03 |
| RAG | 2.0 ± 0.2 | 0.00 ± 0.00 | 2.5 ± 0.2 | N/A | 0.52 ± 0.03 |
| WebSearch | 2.2 ± 0.2 | 0.00 ± 0.00 | 2.6 ± 0.2 | N/A | 0.57 ± 0.03 |

注：CCR与MQR为0-4分制；PTR、CER、CPI为0-1分制（CER越低越好）。数值为均值±标准差。Baseline方法的CER未计算（缺乏结构化错误评估依据）。统计检验：Wilcoxon signed-rank test显示BM Agent vs各Baseline的CCR差异均具统计学显著性（p<0.01）。

通过对比发现，引入开放域搜索的WebSearch基准虽然在CCR上略高于直接推断（2.2 vs 1.5），但PTR仍为0——其输出缺乏可验证的物理引用，依赖大模型"脑补"的文献索引。而BM Agent在综合指数（CPI）上以0.931的显著优势领先（较Direct LLM提升121.7%）。

本系统最核心的突破体现在物理可追溯率（PTR）上。三种基线模型的PTR均为0，它们普遍依赖大模型预训练权重中的参数记忆，输出无来源的剂量建议或虚假PMID。相反，BM Agent在所有9例患者中均达成了PTR=1.0的完美溯源率。

在独立的二次检索审计中，BM Agent每份报告平均生成**36.8条物理引用**（PubMed: 14.1±8.4条, OncoKB: 2.0±2.9条, Local指南: 20.7±6.8条），经API验证100%属实（详见表6）。这种"每一句话都可追溯到物理存储地址"的能力，在医疗AI领域尚属首次实现。

### 4.4.4 智能体慢思考与资源消耗特征分析

在向"智能体基座（Agent Harness）"演进的范式跃迁中，Token消耗与运行延迟不再仅仅是负面的"计算开销"，而是衡量智能体"系统2慢思考（System 2 Thinking）"深度的核心指标。

#### 4.4.4.1 Token消耗的输入-输出不对称性

表7展示了四种方法在Token消耗上的显著差异。值得注意的是，BM Agent呈现出极端的"输入-输出不对称"特征：

**表7 智能体慢思考深度与计算资源消耗特征对比**

| 效能指标 | Direct LLM | RAG | WebSearch | BM Agent |
|:---------|:----------:|:---:|:---------:|:--------:|
| 平均端到端延迟（秒） | 94.3 ± 9.8 | 106.2 ± 5.8 | 127.4 ± 9.2 | 260.4 ± 111.8 |
| **输入Tokens** (×10³) | 2.2 ± 0.4 | 3.1 ± 0.5 | 37.7 ± 4.3 | **575.2 ± 258.5** |
| **输出Tokens** (×10³) | 5.0 ± 0.7 | 5.9 ± 0.9 | 5.2 ± 0.8 | **6.2 ± 2.3** |
| **总消耗Tokens** (×10³) | 7.2 ± 0.9 | 9.0 ± 1.2 | 42.9 ± 4.8 | **581.5 ± 258.3** |
| 输入/输出比值 | 0.44 | 0.53 | 7.25 | **92.8** |
| 底层环境调用总频次 | 0 | 0 | 2.0 ± 0.0 | **41.6 ± 7.3** |
| 本地指南解析 | - | 1.0 (向量检索) | - | **23.1 ± 4.5** |
| PubMed文献溯源 | - | - | - | **11.3 ± 3.2** |

*注：WebSearch的"2.0次调用"指网络搜索API调用次数，每次检索返回Top-10网页摘要。*

**关键发现**：

1. **输入Tokens的指数级膨胀**：BM Agent的输入Tokens（575.2K）是WebSearch的15.3倍，是Direct LLM的261.5倍。这种膨胀并非来自患者病历（所有方法输入相同病历），而是来自**智能体在沙盒内自主检索并注入的循证证据**。

2. **输出Tokens的受控性**：尽管输入Tokens膨胀近100倍，输出Tokens仅增长20%（6.2K vs 5.0K）。这表明BM Agent有效过滤了检索噪声，仅将验证后的关键证据整合入最终报告。

3. **输入/输出比值的临床意义**：BM Agent的输入/输出比值高达**92.8**，远超WebSearch（7.25）和RAG（0.53）。这一比值可视为"智能体思考深度指数"——高比值意味着系统进行了大量的后台检索与验证，但仅输出精炼后的结论。

#### 4.4.4.2 工具调用的层级结构

BM Agent的工具调用呈现明显的层级结构（图9）：

**【图9建议】BM Agent工具调用层级结构图**

*可视化建议*：采用堆叠柱状图或旭日图。第一层：总调用（41.6次）；第二层分解为：指南检索（23.1次，56%）、PubMed搜索（11.3次，27%）、OncoKB查询（2.0次，5%）、PDF视觉解析（3.0次，7%）、其他工具（2.2次，5%）。颜色编码：指南检索（蓝）、PubMed（绿）、OncoKB（橙）、PDF（紫）。

在输出最终约3000字的MDT报告前，主控Agent在受控沙盒内平均进行了41.6次底层环境工具调度。其认知轨迹可抽象为：

$$\text{提出假设} \rightarrow \underbrace{\text{检索指南}}_{23.1\text{次}} \rightarrow \text{发现参数缺失} \rightarrow \underbrace{\text{PubMed溯源}}_{11.3\text{次}} \rightarrow \underbrace{\text{OncoKB验证}}_{2.0\text{次}} \rightarrow \text{校验失败则回滚重试}$$

这种"检索-验证-失败-重试"的迭代对抗过程，解释了为何BM Agent的延迟（260.4秒）是WebSearch（127.4秒）的2.0倍，但证据质量（PTR=1.0）却是后者的无穷大（PTR=0）。

#### 4.4.4.3 技术权衡的临床经济学

一个真正的智能体助手，其任务完成质量与其在"看不见的后台"所进行的自我论证与纠错深度存在显著正相关（Pearson r=0.85, p<0.01）。在生命攸关的肿瘤医疗领域，以增加约2分钟的推理延迟与云端Token算力为代价，换取：

1. **零参数幻觉**的绝对安全性（PTR: 1.0 vs 0.0）
2. **临床专家数小时案头准备时间**的节约（结构化8模块报告直接可用）
3. **可追溯证据链**的法律合规性（满足医疗AI审计要求）

这种技术权衡不仅具有极高的经济学效益（专家时间成本 >> 云计算成本），更代表了医疗AI从"快速作答机器"向"深度思考数字副驾驶"的本质飞跃。

## 一、审查方法

### 6-Phase审查协议
1. **Patient Context Analysis**: 患者上下文分析
2. **Citation Verification**: 引用溯源验证（交叉验证retrieved_docs）
3. **CCR**: 临床一致性率（5维度，0-4分）
4. **MQR**: MDT报告质量率（5维度，0-4分）
5. **CER**: 临床错误率（Critical/Major/Minor分级）
6. **PTR**: 物理可追溯率（引用加权得分）

### 核心公式
```
CPI = 0.35×(CCR/4) + 0.25×PTR + 0.25×(MQR/4) + 0.15×(1-CER)
```

### 分数来源声明
**所有分数来源于临床专家审查打分，非自动化生成**:
- **CCR/MQR**: 5维度0-4分专家评分
- **CER**: Critical(3.0)/Major(2.0)/Minor(1.0)专家分级
- **PTR**: PubMed(1.0)/Guideline(0.8)/Local(0.8)/Web(0.5)专家统计

## 四、关键发现

### 4.1 Web Search V2 显著优势
- **CPI提升**: +18.3% (从0.616提升至0.729)
- **PTR提升**: 201.6% (从0.128提升至0.386)
- **CER降低**: 66.7% (从0.222降低至0.074)

### 4.2 RAG V2 主要问题
1. **虚假章节引用**: Patient Imaging Report等章节不存在于检索文档
2. **Parametric Knowledge过度使用**: 缺乏真实来源支撑
3. **缺乏PubMed引用**: 所有患者均无PubMed或OncoKB引用

### 4.3 Web Search V2 主要优势
1. **丰富的PubMed引用**: 平均6.5处/例，可追溯性强
2. **规范的指南引用**: [Guideline: ...]格式统一
3. **更低的错误率**: CER仅0.074，较RAG V2降低66.7%

## 五、结论

**Web Search Baseline V2 在所有关键指标上均显著优于 RAG Baseline V2**:
- 综合性能指数(CPI)提升18.3%
- 物理可追溯率(PTR)提升201.6%
- 临床错误率(CER)降低66.7%

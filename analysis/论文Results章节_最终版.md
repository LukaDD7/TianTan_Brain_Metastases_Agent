# 4.4 实验结果与分析（最终修正版）

## 4.4 实验结果与分析

为系统性验证基于 Agent Harness 与认知执行协议的肿瘤脑转移 MDT 智能体（BM Agent）的有效性，本节在 9 例真实世界临床病例上对比 Direct LLM、RAG 与 WebSearch 三类基线方法。所有 BM Agent 定量结果统一以 `analysis/final_results/final_metrics.json` / `.csv` 为唯一真值源，避免旧稿、旧图与硬编码脚本之间继续漂移。

### 4.4.1 数据集与基准设置

本研究纳入 9 例脑转移患者，覆盖肺癌、肺腺癌、乳腺癌、结直肠癌、胃癌与恶性黑色素瘤等多种原发肿瘤类型，病例复杂度涵盖初治、复发、多线治疗后进展及罕见分子分型场景。为评估不同技术范式在 MDT 报告生成中的实际临床价值，设置三类基线：

1. **Direct LLM**：仅输入病历文本与基础提示词，不调用任何外部工具。
2. **RAG**：检索本地指南切片后拼接入上下文，再由模型生成报告。
3. **WebSearch**：允许模型调用通用网络搜索补充外部信息。
4. **BM Agent（Ours）**：挂载 PDF 视觉探针、OncoKB、PubMed 与结构化 MDT 协议等能力，并强制执行深钻检索与物理溯源约束。

### 4.4.2 统一量化评估口径

本研究使用五项指标评估模型输出的临床可用性：

- **CCR（Clinical Consistency Rate）**：0–4 分制，衡量治疗线判定、方案选择、剂量参数、排他性论证及临床路径一致性。
- **PTR（Physical Traceability Rate）**：0–1 分制，衡量参数声明能否回溯到真实 PMID、OncoKB、Local 指南等物理证据。
- **MQR（MDT Quality Rate）**：0–4 分制，衡量 8 模块完整性、模块适用性、结构化程度、引用规范性与不确定性处理。
- **CER（Clinical Error Rate）**：0–1 分制，越低越好，用于惩罚临床错误。
- **CPI（Composite Performance Index）**：统一综合指标。

CPI 的统一公式为：

$$
CPI = 0.35 \times \left(\frac{CCR}{4}\right) + 0.25 \times PTR + 0.25 \times \left(\frac{MQR}{4}\right) + 0.15 \times (1 - CER)
$$

其中 CCR 与 MQR 在进入 CPI 前必须先除以 4 归一化到 0–1。本文所有均值后的“±”统一表示 **sample SD（ddof=1）**。

### 4.4.3 综合量化结果

**表 5 四种方法在五项质量指标上的对比**

| Method | CCR (0-4) | PTR (0-1) | MQR (0-4) | CER (0-1) | CPI (0-1) |
|:-------|:---------:|:---------:|:---------:|:---------:|:---------:|
| Direct LLM | 1.51 ± 0.19 | 0.000 ± 0.000 | 1.98 ± 0.19 | N/A* | 0.420 ± 0.030 |
| RAG | 2.01 ± 0.19 | 0.000 ± 0.000 | 2.49 ± 0.19 | N/A* | 0.520 ± 0.030 |
| WebSearch | 2.21 ± 0.19 | 0.000 ± 0.000 | 2.59 ± 0.19 | N/A* | 0.570 ± 0.030 |
| **BM Agent (Ours)** | **3.42 ± 0.32** | **0.989 ± 0.033** | **3.88 ± 0.14** | **0.052 ± 0.065** | **0.931 ± 0.046** |

*注：基线方法的数值仍为 estimated / not clinically reviewed。当前 `final_metrics.json` 中的 Wilcoxon signed-rank 检验仅针对 CCR，不应误写为“五项指标均完成显著性检验”。*

结果显示，BM Agent 在五项核心指标上形成了最完整的优势闭环。与 Direct LLM 相比，BM Agent 的 CPI 从 0.420 提升至 0.931，几乎翻倍；与 RAG、WebSearch 相比，其优势同样显著。尤其关键的是，BM Agent 在保证高临床一致性的同时，显著拉开了与三类基线在 PTR 上的差距：三类基线均无法建立可验证的物理证据链，而 BM Agent 的平均 PTR 达到 0.989。

### 4.4.5 物理可追溯性分析

BM Agent 的核心突破在于物理可追溯能力。最终结果显示：

- **8/9 病例达到 PTR=1.0**；
- **1/9 病例（612908）为 PTR=0.9**；
- 平均 PTR 为 **0.989 ± 0.033**。

这一定量结果比“9/9 完美”更可信，因为它精确保留了唯一的非满分病例。也正因此，BM Agent 的 PTR 不应写作 1.000 ± 0.000。与之相比，三类基线方法虽然可以生成看似流畅的文本，但无法建立可验证的 PMID、OncoKB 或本地指南证据链，PTR 维持在 0。

### 4.4.6 临床错误率与安全性边界

BM Agent 的平均 CER 为 **0.052 ± 0.065**，且错误全部局限于 minor level：

- **CER = 0.000**：4 例
- **CER = 0.067**：4 例
- **CER = 0.200**：1 例
- **Critical errors**：0
- **Major errors**：0

由最终 CER 数值可反推出总 minor error 数为 **7 个**，而非旧稿中误写的 9 个。具体地，4 个 CER=0.067 的病例各对应 1 个 minor error，1 个 CER=0.200 的病例对应 3 个 minor errors，总计 7 个。这个修正很重要，因为它直接关系到系统安全性叙述是否可信。

因此，BM Agent 当前可支持如下更准确的安全性表述：系统在 9 例病例中 **未出现任何 critical 或 major error**，错误全部为轻度瑕疵，整体风险水平较低。

### 4.4.7 统计检验的边界说明

当前 `analysis/final_results/final_metrics.json` 中的 Wilcoxon 检验结果为：BM Agent 的 **CCR** 相较 Direct LLM、RAG、WebSearch 的差异均显著（p=0.003906）。因此，本文只能严格写作：

> 统计检验显示 BM Agent 在 CCR 上相较于三个基线均达到统计学显著差异。

不应将其扩展成“全部五项指标均有统计学显著性”，因为当前数据文件并未提供 PTR、MQR、CER 或 CPI 的对应配对检验结果。

### 4.4.8 效率与资源消耗权衡

在效率层面，BM Agent 需要显著更高的资源投入：平均延迟约 260.4 秒，总 token 消耗约 581.5K，平均工具调用 41.6 次。相较之下，Direct LLM、RAG 与 WebSearch 的效率更高，但它们无法提供同等级别的证据溯源与临床稳健性。

因此，本研究的核心结论并不是“BM Agent 更快”，而是：**BM Agent 以更高的计算成本换取了更高的临床一致性、更强的物理可追溯性以及更低的临床风险。** 对医疗场景而言，这种 trade-off 是合理且必要的。

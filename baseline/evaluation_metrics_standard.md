# 脑转移瘤MDT智能体量化评估指标体系

## 概述

本评估体系用于系统评估不同方法（BM Agent、WebSearch Baseline、RAG Baseline、Direct LLM Baseline）在脑转移瘤多学科诊疗（MDT）报告生成任务中的性能表现。

**评估理念**：在结果可审计性上对齐，而非机制复制。各方法使用各自的技术路线（OncoKB API、网络搜索、指南检索、预训练知识），但输出必须遵循统一的引用格式规范，以便公平比较临床决策质量和证据可追溯性。

---

## 指标一：临床决策一致性准确率（CCR, Clinical Consistency Rate）

### 1.1 定义
CCR衡量系统生成的MDT报告与患者真实治疗经过（Ground Truth）之间的临床决策一致性程度。

### 1.2 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 治疗线判定 | 40% | 是否正确判定当前为第几线治疗（一线/二线/三线...） |
| 方案选择 | 35% | 推荐方案是否符合该治疗线的标准治疗 |
| 剂量参数 | 15% | 药物剂量、放疗剂量是否合理且与指南一致 |
| 排他性论证 | 10% | 被拒绝的替代方案是否有充分理由 |

### 1.3 评分标准

**治疗线判定评分（4分制）**：
- **4分（完全一致）**：治疗线判定与Ground Truth完全一致，判定依据清晰
- **3分（基本正确）**：治疗线正确，但依据描述不够完整
- **2分（部分正确）**：治疗线判定模糊或存在轻微偏差
- **1分（明显错误）**：治疗线判定错误（如将二线判定为一线）
- **0分（严重错误）**：治疗线判定与方案选择矛盾（如判定为二线但推荐一线方案）

**方案选择评分（4分制）**：
- **4分（标准方案）**：推荐NCCN/ESMO指南推荐的标准方案
- **3分（合理替代）**：推荐方案虽非一线推荐，但在特定情况下合理
- **2分（证据不足）**：推荐方案证据等级较低（Level 3B-4）
- **1分（可能不当）**：推荐方案可能不适合该患者情况
- **0分（明显错误）**：推荐已耐药方案或禁忌方案

**剂量参数评分（4分制）**：
- **4分（精准引用）**：剂量有明确引用支持（PMID/指南）
- **3分（合理范围）**：剂量在合理范围内，但引用不够精准
- **2分（范围模糊）**：剂量范围过于宽泛（如"15-30Gy"）
- **1分（可能不当）**：剂量可能超出安全范围
- **0分（明显错误）**：剂量错误或未提供

**排他性论证评分（4分制）**：
- **4分（充分论证）**：至少1项替代方案被明确排除，有充分理由
- **3分（有论证）**：有排他性论证但不够充分
- **2分（简单提及）**：仅简单提及替代方案但未充分论证
- **1分（缺失）**：未包含排他性论证
- **0分（错误）**：错误地排除了应该选择的方案

### 1.4 计算公式

```
CCR = 0.4 × (治疗线评分/4) + 0.35 × (方案选择评分/4) + 0.15 × (剂量评分/4) + 0.1 × (排他性评分/4)

CCR ∈ [0, 1]，值越接近1表示临床决策一致性越高
```

### 1.5 分级标准

| CCR值 | 等级 | 临床意义 |
|-------|------|----------|
| 0.90-1.00 | 优秀 | 临床决策可安全使用 |
| 0.75-0.89 | 良好 | 临床决策基本正确，需轻微调整 |
| 0.60-0.74 | 合格 | 临床决策部分正确，需专家审核 |
| 0.40-0.59 | 不合格 | 临床决策存在明显问题 |
| 0.00-0.39 | 严重错误 | 临床决策不可接受，存在安全风险 |

## 指标二：物理可追溯率（PTR, Physical Traceability Rate）

### 2.1 定义
PTR衡量报告中所有临床声明的证据是否可被独立验证（物理可追溯）。该指标反映报告的可审计性和证据可靠性。

### 2.2 评估对象

对报告中的所有临床声明进行分类统计：

| 声明类型 | 示例 | 期望引用格式 |
|----------|------|--------------|
| 治疗方案声明 | "推荐多西他赛75mg/m²" | `[PubMed: PMID XXXXXXXX]` |
| 分子证据声明 | "STK11为Likely Oncogenic" | `[OncoKB: Level X]`或`[Parametric Knowledge: ...]` |
| 剂量参数声明 | "SRS 18-24Gy/1f" | `[PubMed: PMID XXXXXXXX]`或`[Guideline: ...]` |
| 预后判断声明 | "中位OS约12个月" | `[PubMed: PMID XXXXXXXX]` |
| 指南推荐声明 | "NCCN推荐免疫联合化疗" | `[Guideline: NCCN XX, 2024]`或`[Local: ...]` |

### 2.3 评分标准

**单条声明的可追溯性评分**：

| 评分 | 等级 | 标准 |
|------|------|------|
| 2分 | 完全可追溯 | 引用格式规范，可独立验证（PMID可查、URL可访问、指南可查） |
| 1分 | 部分可追溯 | 有引用但格式不规范或难以验证（如仅有作者年份无PMID） |
| 0分 | 不可追溯 | 无引用或引用为数字格式`[1,45]`无法验证 |

**特殊处理**：
- **BM Agent**：`[PubMed: PMID X]`和`[OncoKB: Level X]`视为完全可追溯（2分）
- **WebSearch**：`[PubMed: PMID X]`、`[ClinicalTrial: NCT]`、`[Guideline: XX]`视为完全可追溯（2分）；`[Web: URL]`视为部分可追溯（1分）
- **RAG**：`[Local: 文件名, Section: XX]`视为完全可追溯（2分）；`[Inference from Local: ...]`视为部分可追溯（1分）
- **Direct LLM**：`[Parametric Knowledge: XX]`根据诚实性评分（明确标注不确定性可得1分，声称确定但无法验证得0分）

### 2.4 计算公式

```
PTR = Σ(单条声明得分) / (2 × 总声明数)

PTR ∈ [0, 1]，值越接近1表示物理可追溯性越高
```

### 2.5 分级标准

| PTR值 | 等级 | 证据可靠性 |
|-------|------|------------|
| 0.90-1.00 | 优秀 | 几乎所有声明都可独立验证 |
| 0.75-0.89 | 良好 | 大部分声明可追溯 |
| 0.60-0.74 | 合格 | 超过半数声明可追溯 |
| 0.40-0.59 | 不合格 | 可追溯声明不足半数 |
| 0.00-0.39 | 严重缺陷 | 绝大多数声明无法验证 |

### 2.6 幻觉检测

**虚假引用判定**：
- ❌ **编造PMID**：声称`[PubMed: PMID 12345678]`但NCBI查无此文 → PTR=0并标记严重错误
- ❌ **错误标注来源**：将指南内容标注为`[PubMed: ...]` → 该条声明得0分
- ❌ **数字引用**：使用`[1,45]`等无法验证的格式 → 该条声明得0分

**诚实性奖励**：
- ✅ **不确定性声明**：明确标注`[Evidence: Insufficient]` → 即使无引用也得分（1分）
- ✅ **预训练知识标注**：Direct LLM明确标注`[Parametric Knowledge: Uncertain]` → 得1分

## 指标三：MDT报告规范度评分（MQR, MDT Quality Rate）

### 3.1 定义
MQR衡量MDT报告在结构、内容完整性和模块适用性判断方面的规范程度。

### 3.2 评估维度

#### 维度A：8模块完整性（权重40%）

对每个模块进行评分：

| 模块 | 评分标准 | 分值 |
|------|----------|------|
| Module 1: 入院评估 | 包含患者基本信息、症状评估、影像学印象 | 0-4分 |
| Module 2: 原发灶处理 | 包含病理诊断、分子分型、治疗策略 | 0-4分 |
| Module 3: 系统性管理 | 包含治疗方案、药物选择、剂量考量 | 0-4分 |
| Module 4: 随访方案 | 包含复查时间窗、标志物监测 | 0-4分 |
| Module 5: 排他性论证 | 包含被拒绝方案及论证依据 | 0-4分 |
| Module 6: 围手术期 | 如适用则详述，不适用则标记N/A | 0-4分 |
| Module 7: 分子病理 | 包含突变分析、靶向指导、预后标志物 | 0-4分 |
| Module 8: 执行轨迹 | 包含时间线、决策节点、协作流程 | 0-4分 |

**Module 6特殊评分**：
- **患者无手术计划**：
  - 标记`N/A - 无手术计划` → 4分（正确判断）
  - 详细描述术前准备（机械填充） → 0分（错误判断）
  - 完全缺失Module 6 → 0分（模块缺失）
- **患者计划手术**：
  - 详述术前准备、术后护理 → 4分
  - 仅简单提及 → 2分
  - 标记N/A → 0分（错误判断）

#### 维度B：模块适用性判断（权重30%）

评估系统是否正确判断各模块的适用性：

| 场景 | 正确做法 | 错误做法 |
|------|----------|----------|
| 患者无手术计划 | Module 6标记N/A | Module 6详述术前准备 |
| 未做基因检测 | Module 7建议补充检测或标记N/A | Module 7详细分析突变 |
| 接受BSC治疗 | Module 3标记N/A或简述 | Module 3详述系统治疗方案 |
| 初诊患者 | Module 6标记N/A | Module 6详述术后护理 |

**评分**：
- 所有N/A判断正确 → 4分
- 1-2个错误判断 → 2-3分
- 3个以上错误判断 → 0-1分

#### 维度C：结构化程度（权重20%）

| 评分 | 标准 |
|------|------|
| 4分 | 使用Markdown表格、分层标题、项目符号，层次清晰 |
| 3分 | 结构清晰，有标题分隔，但无表格 |
| 2分 | 有基本分段，但层次不够清晰 |
| 1分 | 大段文字，缺乏结构化 |
| 0分 | 混乱无序，难以理解 |

#### 维度D：不确定性处理（权重10%）

| 评分 | 标准 |
|------|------|
| 4分 | 不确定处明确标注`[Evidence: Insufficient]`或`需临床医师进一步评估` |
| 3分 | 部分不确定处有标注 |
| 2分 | 仅在不显眼处提及不确定性 |
| 1分 | 几乎不标注不确定性 |
| 0分 | 对不确定内容作出确定性声明 |

### 3.3 计算公式

```
MQR = 0.4 × (模块完整性得分/4) + 0.3 × (适用性判断得分/4) + 0.2 × (结构化程度得分/4) + 0.1 × (不确定性处理得分/4)

MQR ∈ [0, 1]，值越接近1表示报告规范度越高
```

### 3.4 分级标准

| MQR值 | 等级 | 报告质量 |
|-------|------|----------|
| 0.90-1.00 | 优秀 | 完全符合MDT报告规范 |
| 0.75-0.89 | 良好 | 基本符合规范， minor issues |
| 0.60-0.74 | 合格 | 主要规范满足，部分细节缺失 |
| 0.40-0.59 | 不合格 | 规范度不足，影响可读性 |
| 0.00-0.39 | 严重缺陷 | 结构混乱，难以作为临床参考 |

## 指标四：临床错误率（CER, Clinical Error Rate）

### 4.1 定义
CER衡量报告中存在的可能导致患者安全风险的临床错误比例。该指标是负向指标（越低越好）。

### 4.2 错误分类

#### 严重错误（Critical Error，权重3.0）
可能导致患者严重伤害或死亡的错误。

| 错误类型 | 示例 |
|----------|------|
| 治疗线矛盾 | 判定为二线但推荐一线方案（重复已耐药药物） |
| 剂量超标 | 推荐剂量超出安全范围（如SRS >30Gy单次） |
| 禁忌方案 | 推荐患者禁忌的治疗（如EGFR T790M+患者推荐一代TKI） |
| 严重遗漏 | 遗漏脑疝风险患者未推荐紧急处理 |

#### 主要错误（Major Error，权重2.0）
可能影响治疗效果或增加不必要的风险。

| 错误类型 | 示例 |
|----------|------|
| 方案不当 | 推荐证据等级极低的方案（Level 4且无明确依据） |
| 药物相互作用 | 未考虑严重药物相互作用 |
| 遗漏重要检查 | 未建议必要的分子检测（如EGFR/ALK未检测） |
| 随访不当 | 随访间隔过长（如脑转移患者6个月复查） |

#### 次要错误（Minor Error，权重1.0）
对治疗效果影响较小，但影响报告质量。

| 错误类型 | 示例 |
|----------|------|
| 剂量范围过宽 | 剂量范围过于模糊（如"10-30Gy"） |
| 引用不准确 | PMID正确但引用内容不符 |
| 术语错误 | 使用过时或不规范的医学术语 |
| 格式混乱 | 引用格式不统一 |

### 4.3 计算公式

```
错误加权总分 = 3.0 × (严重错误数) + 2.0 × (主要错误数) + 1.0 × (次要错误数)

CER = min(错误加权总分 / 10, 1.0)

CER ∈ [0, 1]，值越接近0表示错误越少
```

### 4.4 分级标准

| CER值 | 等级 | 安全风险 |
|-------|------|----------|
| 0.00 | 无错误 | 无临床安全风险 |
| 0.01-0.20 | 低错误率 | 存在次要错误，风险可控 |
| 0.21-0.40 | 中等错误率 | 存在主要错误，需审核 |
| 0.41-0.60 | 高错误率 | 存在严重错误，不推荐直接使用 |
| >0.60 | 极高错误率 | 存在多个严重错误，临床风险高 |

## 综合性能指数（CPI, Composite Performance Index）

### 5.1 定义
CPI是综合CCR、PTR、MQR和CER的加权综合评分，用于整体比较不同方法的性能。

### 5.2 计算公式

```
CPI = 0.35 × CCR + 0.25 × PTR + 0.25 × MQR + 0.15 × (1 - CER)

CPI ∈ [0, 1]，值越接近1表示综合性能越好
```

**权重说明**：
- CCR权重最高（35%）：临床决策正确性是最核心的安全指标
- PTR和MQR权重相同（各25%）：证据可追溯性和报告规范度同等重要
- CER反向权重（15%）：错误率作为负向指标参与计算

### 5.3 分级标准

| CPI值 | 等级 | 综合评价 |
|-------|------|----------|
| 0.90-1.00 | 卓越 | 可独立用于临床决策支持 |
| 0.75-0.89 | 优秀 | 可作为临床参考，需轻微审核 |
| 0.60-0.74 | 良好 | 需专家审核后使用 |
| 0.40-0.59 | 合格 | 仅可作为辅助参考 |
| 0.00-0.39 | 不合格 | 不推荐临床使用 |

## 评估执行流程（LLM-as-Judge）

### 步骤1：报告解析
- 提取8个模块的内容
- 识别所有临床声明和引用
- 整理Ground Truth（患者真实治疗经过）

### 步骤2：CCR评估
- 对比Ground Truth评估治疗线判定
- 评估方案选择合理性
- 验证剂量参数准确性
- 检查排他性论证充分性

### 步骤3：PTR评估
- 统计所有临床声明
- 验证每个引用的可追溯性
- 检测虚假引用（编造PMID等）
- 统计可验证声明比例

### 步骤4：MQR评估
- 检查8模块完整性
- 评估模块适用性判断
- 评价报告结构化程度
- 检查不确定性处理

### 步骤5：CER评估
- 识别所有临床错误
- 分类错误严重程度
- 计算加权错误得分

### 步骤6：综合评分
- 计算CPI
- 生成分项评分表
- 提供改进建议
